# AIS-016 — CONTRIBUTING note: maintainers track merged skills in harness (drift prevention, Layer 3)

**Status:** done (2026-07-22)

## What shipped
[CONTRIBUTING.md](../../CONTRIBUTING.md) additions:
- A **"Maintainers only — work tracking"** callout under *How to contribute*: the repo tracks
  work in `harness/` (private feature list + notes); contributors don't touch it; the maintainer
  adds the harness entry when merging a new/changed skill; `bash init.sh` warns on any skill with
  no harness reference (ties Layer 3 → Layer 2 / AIS-014).
- A new *maintainer* item in the review checklist: "the merged skill is tracked in `harness/` —
  `bash init.sh` parity check is quiet."

## Why a doc note (not enforcement) for the contributor path
External contributors can't and shouldn't manage the maintainer's internal work-control. So the
external-PR path is handled by (a) init.sh catching drift next session (AIS-014) and (b) this
written expectation that the maintainer records the entry at merge — not by blocking the PR.
Pure docs; no push required to be effective for the maintainer reading it.
