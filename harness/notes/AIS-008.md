**Status:** done (2026-07-22)

## What shipped
`skills/skill-publisher/SKILL.md` — tool-neutral meta-skill implementing UC4 (publish flow):
Step 0 ask-first → Step 1 audit (markdown-only, no secrets/personal/internal, tool-neutral, no
dangerous instructions) → Step 2 normalize (frontmatter + standard footer) → Step 3 regenerate
the registry via the repo's own method (never hand-edit) → Step 4 publish (owner push / fork+PR,
Co-Authored-By) → Step 5 stamp provenance + report every check. Frontmatter + footer present;
added to `registry.json` (now 7 skills); CI green.

## Gotcha (worth remembering)
First version's audit checklist wrote the literal `/Users/…` / `C:\…` as examples-to-avoid.
The CI + init.sh scan for `/Users/` and `C:\` are intentionally blunt greps over `skills/`, so
they FAILED on it (CI run 29895083850 = failure). Reworded to avoid the literal tokens (macOS/
Linux home dirs, Windows drive paths) → CI green (29895363696). Lesson: a skill must not contain
`/Users/` or `C:\` even as illustrative text; use generic phrasing.
