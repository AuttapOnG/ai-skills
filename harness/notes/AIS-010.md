## Key finding
**The INSTALL protocol is tool-neutral.** Codex, given only `README.md`, correctly performed
every step and — notably — chose `AGENTS.md` as the tool-neutral global-instructions file for
the publish-line (no Claude-specific assumption). The discovery-ladder phrasing worked.

## Protocol feedback surfaced by the cross-tool test (README polish candidates, non-blocking)
Codex followed every step, but flagged:
- INSTALL "fetch the skill folder" gives a raw-file base but no way to LIST a folder's files;
  an agent must discover contents via an available API — slightly underspecified.
- Provenance "next to the installed skill folder" is ambiguous (inside vs sibling) → clarify to
  "inside the skill folder".
- Destination skills-dir + global-instructions file are necessarily runtime-specific (by design).
