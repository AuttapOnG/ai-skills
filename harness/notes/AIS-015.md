## What changed (local)
[skills/skill-publisher/SKILL.md](../../skills/skill-publisher/SKILL.md) gained a new
**Step 4 — Update the project's work-control memory (if it has one)** (Publish renumbered 4→5,
Stamp provenance 5→6). It tells the publisher: if the target repo tracks work in a machine-readable
task memory (harness / feature list / backlog / referenced issue tracker), record the skill there
**in the same change** so registry and task memory don't drift; discover the format from
CONTRIBUTING / README / an existing entry; skip if the repo has none. Step 6's verify checklist
gained "work-control entry added/updated (if the repo tracks one)".

## Tool-neutrality
The step is conditional and format-agnostic — no `AIS-NNN`, no hardcoded `harness/feature_list.json`
as a requirement (only as an example). So the skill stays valid when installed into a repo with no
harness. This closes the **self-publish** path (the case that produced the skill-updater gap);
external-PR path is covered by AIS-014 (init.sh guard) + AIS-016 (CONTRIBUTING).
