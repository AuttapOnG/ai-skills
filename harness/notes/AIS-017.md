# AIS-017 — LICENSE (MIT)

**Status:** done (2026-07-22)

## What shipped
Root [`LICENSE`](../../LICENSE) — MIT, `Copyright (c) 2026 Auttapong Tura`.

## Why it was needed (not optional)
The repo is a **public install/contribute registry**, but had no license. A public repo with
no license = **all rights reserved by default** — nobody may legally copy, install, modify, or
redistribute the content. That directly contradicts the README INSTALL protocol ("fetch the
skill folder and drop it into your tool"). MIT makes the intended reuse lawful.

## Choices
- **MIT** over CC BY 4.0 / Apache-2.0 (user decision): lowest friction for an agent-installed
  registry, universally recognized, GitHub renders it. The standard skill footer already carries
  attribution back to the registry as a courtesy, so MIT's minimal terms suffice.
- Holder = `Auttapong Tura` (matches the repo-local git identity); year 2026.
- **Name only, no email** — a copyright name is expected/public; keeps with the repo's
  personal-data hygiene (the maintainer name is already the git author throughout history).

## Hygiene / CI
- LICENSE is at repo root → not under `skills/`, so init.sh's path scan and CI's
  `skills/`-scoped checks are unaffected. Markdown-only rule is unaffected (LICENSE is plain text).
- Inbound = outbound stated in README's new License section (AIS-018).
