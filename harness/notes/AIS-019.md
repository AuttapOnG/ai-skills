# AIS-019 — README "Quick start (for humans)" section

**Status:** done (2026-07-23)

## What shipped
One addition to [`README.md`](../../README.md): a **Quick start (for humans)** section placed
after the intro paragraphs and before the agent-addressed blockquote. Content:
- States the human runs nothing themselves — their AI agent does (tool-neutral: Claude Code,
  Codex, Gemini CLI named only as examples).
- Two copy-paste prompts (install / update) whose wording matches the protocol trigger
  phrases already declared in INSTALL ("install skill(s) from this repo") and UPDATE
  ("update my skills from the repo") — no new trigger surface for agents to interpret.
- Notes the agent reports a verification checklist back (also explains the `- [ ]` Verify
  checkboxes a human sees in the protocols), and that everything below the section is
  addressed to the agent.

## Design decisions
- Origin: user observation (2026-07-23) that the README had no guide for humans — the whole
  file was addressed to AI agents, so a person landing on the repo had no next action.
- Approaches considered: quick-start section on top of README (chosen) · separate USAGE.md
  (extra click + drift surface) · human-first README rewrite with protocols moved to
  PROTOCOL.md (violates spec §5 "README = protocol" and risks breaking skill-updater /
  skill-publisher / CONTRIBUTING references).
- The agent-addressed blockquote below the new section is kept **verbatim** — protocols and
  their Verify checklists untouched, so no re-test of UC1/UC2 needed.
- Spec §5 amended with a matching bullet (same commit) so the outline stays the source of truth.
