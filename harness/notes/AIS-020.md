# AIS-020 — orchestrate skill (cost-optimized subagent routing)

**Status:** done (2026-07-23)

## What shipped
[`skills/orchestrate/SKILL.md`](../../skills/orchestrate/SKILL.md) — turns the main session
into a cost-optimized dispatcher: decompose → route each task to the cheapest capable tier →
recon/distill before escalating → capped return contract; serial dispatch by default with an
explicit user opt-in to parallel. registry.json regenerated (10 skills).

## Origin & IP hygiene
The *concept* (route-by-cheapest-capable + context-volume routing) was seen in a
company-internal, Claude-specific skill. **No text was reused** — this is a from-scratch
rewrite, and nothing in the skill, notes, or commit references the internal source by name
(public-hygiene rule). What was deliberately dropped from the original concept:
- Hardcoded model names/prices in normative text (rots; Claude tiers appear once, as a
  clearly-labeled example only — allowed per README neutrality rule).
- The aggressive proactive trigger ("use on any 3+ subtask request").
- The absolute serial-only mandate (conflicts with runtimes whose guidance favors parallel
  dispatch of independent work).

## Key decisions
- **Explicit-trigger only.** Description says "Use ONLY when the user explicitly asks …
  never proactively." This is the user's chosen calling convention: say "orchestrate" for
  cost-routing inside the runtime; name an external CLI (codex/gemini) for
  [`cli-subagents`](../../skills/cli-subagents/SKILL.md) delegation. Prevents trigger
  collisions between the two skills — the "When NOT to use" section cross-references
  cli-subagents explicitly.
- **Tier-discovery ladder** instead of a fixed model lineup (tool-neutral): discover the
  runtime's delegation mechanism → discover which tiers the account can use → ask the user
  once if unsure; degrade gracefully to context-volume-only when a single tier exists.
- **Serial by default, parallel by consent** — keeps the cost posture (pruning between
  dispatches, low peak, cache-friendly) without contradicting host-runtime parallel guidance.
- Name kept as `orchestrate`: matches the calling convention keyword. Note: anyone who also
  installs a same-named skill from another source into the same skills dir will collide —
  acceptable; the two are same-purpose alternatives, not complements.
